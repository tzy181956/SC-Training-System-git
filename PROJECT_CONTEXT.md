# 项目背景说明

## 项目名称

体能训练管理平台 V1

## 一句话目标

这是一个面向团队体能训练场景的 Web 平台，重点支持教练制定训练计划、分配训练周期、队员在平板上执行训练并记录训练结果。

## 当前技术栈

- 前端：Vue 3
- 后端：FastAPI
- 数据库：SQLite
- ORM：SQLAlchemy
- 图表：ECharts
- 前后端分离

## 当前运行要求

- Python 3.12
- Node.js 18+
- Windows 一键脚本启动

推荐启动方式：

- 双击 `scripts/start_system.bat`

## 当前交互结构

系统包含两种模式：

- 训练模式
- 管理模式

特点：

- 当前为免登录模式
- 默认进入训练模式
- 右上角按钮切换模式
- 系统会记住上次使用的模式

## 当前已完成主流程

- 动作库管理
- 训练模板管理
- 计划分配
- 训练执行
- 自动调重建议
- 测试数据记录
- 训练数据查看

## 核心业务对象

- 运动员
- 动作
- 标签
- 训练模板
- 训练模板动作
- 计划分配
- 训练会话
- 训练动作项
- 组记录
- 测试记录

## 当前训练模式规则

- 支持共享训练模式
- 先选日期，再选人，再选计划，再执行训练
- 训练中每组记录：
  - 实际重量
  - 实际次数
  - RIR
- 系统返回下一组建议
- 重量录入已支持：
  - 快捷点按增减
  - 手动输入

## 当前管理模式规则

- 教练侧可管理：
  - 运动员
  - 动作库
  - 训练模板
  - 计划分配
  - 测试数据
  - 训练数据
- 管理模式强调中文展示和 iPad 横屏可读性

## 当前示例数据特点

示例数据已经覆盖：

- 多个项目与队伍
- 多名运动员
- 多份训练模板
- 百分比负荷与固定负荷
- 测试数据
- 多条训练历史记录
- 训练数据趋势与异常提醒

## 已知设计取舍

- 当前是开发期结构，强调可运行与可扩展
- 暂未引入复杂权限系统
- 暂未接入真实 AI 模型，只保留规则引擎与预留位
- PWA 先做基础版，不包含离线缓存
- 当前数据库初始化脚本面向开发环境，可重建数据库

## 后续开发时应优先保持

- 训练模式触摸优先
- 所有用户界面尽量全中文
- 核心业务逻辑在后端 services
- 不轻易破坏一键启动链路
