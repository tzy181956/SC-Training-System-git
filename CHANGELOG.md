# CHANGELOG

## Unreleased / V1-dev

### Added

- 增加训练模式与训练 session 主流程。
- 增加计划分配概览、分组展示与批量取消分配。
- 增加训练模式下当前组提交、历史组即时修改。
- 增加训练状态颜色提示、自动完成动作、自动完成 session。
- 增加更稳的 Windows 初始化与启动脚本链路。
- 增加运动员主档身体字段：
  - `height`
  - `weight`
  - `body_fat_percentage`
  - `wingspan`
  - `standing_reach`
- 增加测试记录展示字段 `result_text`。
- 增加真实 Excel 数据导入脚本 `backend/scripts/import_real_test_data.py`。
- 增加运行时 schema 同步逻辑 `backend/app/core/schema_sync.py`。
- 增加训练模式顶部访问入口卡片，支持显示局域网访问地址、复制按钮和二维码。

### Changed

- 训练模板彻底删除 `rest_seconds / 休息时间`：前端编辑卡片、保存 payload、后端模型与 schema、运行时数据库列同步全部移除，历史数据随删列一并清理。
- 训练模板和训练展示不再暴露“顺序”输入或编号，统一改为按卡片排列顺序执行与展示。
- 训练模板动作卡片中的“动作备注”输入框收紧为单行高度，减少编辑区占用空间。
- 训练模板动作卡片改为“一级分类 + 二级分类 + 可搜索动作”三段式选动作，并将第二行参数区重排为紧凑布局，训练目标改为自由输入。
- 调整训练模板动作卡片的桌面端栅格断点和列宽比例，避免一级分类、二级分类、动作搜索及参数区过早退化成单列。
- 将训练模板动作卡片的桌面端两行参数区改为稳定的横向工具条布局，确保一级分类、二级分类、动作搜索以及组数/次数/负荷等字段保持同排展示。
- 训练模板动作搜索从原生 `datalist` 改为自定义可搜索下拉，关闭状态统一为类下拉框样式，打开后在同宽面板内搜索和选择动作。
- 训练模式改为“点计划即训练”，不再依赖开始训练前的确认页。
- 当前组恢复为手动确认提交，历史组保持即时修改。
- 训练录入区改成更适合 iPad 的大按钮交互。
- 训练模式左侧队员计划改为常驻展开。
- 训练模式侧边栏新增按日期联动的队伍筛选。
- 测试数据页改为支持展示时间文本和力量体重比计算。
- 运动员页扩展为可编辑身体主档数据。
- 缺少测试基准时，计划分配不再被阻断，改为“训练时录入”。
- 启动脚本的自动初始化链路不再覆盖真实业务数据。
- 真实数据导入脚本改为先要求输入固定确认词，再执行批量清理和导入。
- 共享数据库文件 `backend/training.db` 纳入 Git 跟踪，跨电脑同步改为依赖提交数据库文件。
- 动作库重构为以 Excel `exos_action_library_tagged_for_codex.xlsx` / `动作库_标签版` 为唯一权威源的结构化动作库。
- 动作库页面收口为日常维护模式，网页端只保留浏览、筛选、新建、编辑、删除；导入能力仅保留为维护级脚本和 API。
- 动作库左侧筛选面板改为使用后端 facets 聚合接口，不再由前端从全量动作本地现算。
- 动作库和运动员页的顶部筛选区统一为“搜索框单独一行，分类/筛选控件下一行”的布局。
- 训练模式头部改为紧凑单行布局，访问地址卡片、当前模式和模式切换统一收口到同一行。
- 训练模板新增动作时，`主项动作` 和 `启用自动调重` 默认改为不勾选。
- 训练模板页改为“顶部统一保存”模式，不再要求对单个动作卡片单独点击保存。
- 训练模板页去掉“快速添加动作”表单，改为仅保留“添加动作”按钮，新增后直接在动作卡片内完成选择和填写。
- 动作库页面去掉用户可见的 `EXOS` 字样，只保留内部数据来源说明。
- 删除动作库中的 `负荷类型 / load_profile` 字段，保留真正仍在使用的步长等字段。
- 当前数据库中的运动员性别统一更新为 `女`。
