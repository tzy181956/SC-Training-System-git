# 动作库筛选面板数据来源

本次性能优化后，动作库页面的数据来源拆分为两类：

- `GET /api/exercises`
  - 返回动作结果列表
  - 供中间动作结果区和前端本地筛选使用

- `GET /api/exercises/facets`
  - 返回左侧筛选面板所需的元数据
  - 包括一级分类、二级分类映射和结构化标签候选项

这样做的目的：

- 前端不再从全量动作列表本地推导 facet 候选项
- 避免页面进入时对全部动作重复做全量聚合
- 将“动作结果数据”和“筛选面板元数据”的职责分离

本次只优化左侧标签和分类面板加载性能，不改变现有筛选交互形态，也不引入服务端结果筛选或分页。
