# AGENTS.md

本仓库的协作约束如下：

## 变更记录要求

- 任何会修改仓库内容的工作，都必须同步更新根目录 `CHANGELOG.md`。
- 不允许把 changelog 留到“最后统一补”，每次完成一轮实际改动就要写入本轮变化。
- 如果一次工作包含多个子修改，`CHANGELOG.md` 至少要概括：
  - 新增能力
  - 行为变更
  - 修复项
  - 重要数据或运维说明

## 文档同步要求

- 当功能边界、维护方式、启动方式、数据来源或协作流程发生变化时，必须同步检查并更新：
  - `README.md`
  - `PROJECT_CONTEXT.md`
  - `DEVELOPMENT_GUIDE.md`
  - 相关专项说明文档

## 执行要求

- 把“更新 changelog”视为和“跑构建验证”同级的收尾步骤，不得省略。
- 如果发现 `CHANGELOG.md` 或协作文档出现乱码、过期或与当前实现不符，应优先修复后再继续累积新改动。
